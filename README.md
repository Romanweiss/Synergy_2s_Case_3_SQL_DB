# Кейс-задача №3 — Проектирование базы данных «Туризм» (PostgreSQL)

## Описание
В проекте спроектирована реляционная база данных предметной области «Туризм».
База данных покрывает основные процессы туристического агентства:
- хранение справочной информации (клиенты, туры, услуги, менеджеры);
- оформление заказов туров;
- контроль связей между сущностями через внешние ключи.

## Соответствие условию задания
Требование задания:
- 4–5 таблиц предметной области;
- 3–4 таблицы-справочника;
- 1 таблица переменной информации;
- первичные ключи во всех таблицах;
- связи через внешние ключи из переменной таблицы в справочники.

Выполнено:
- всего таблиц: **5**;
- таблицы-справочники: **4** (`clients`, `tours`, `services`, `managers`);
- таблица переменной информации: **1** (`orders`);
- PK созданы во всех таблицах;
- FK в `orders` ссылаются на PK таблиц-справочников.

## Структура базы данных
### Таблицы-справочники
- `clients` — клиенты туристического агентства.
- `tours` — перечень туров.
- `services` — дополнительные услуги.
- `managers` — сотрудники/менеджеры.

### Таблица переменной информации
- `orders` — заказы туров.

### Связи (FK)
- `orders.client_id` -> `clients.client_id`
- `orders.tour_id` -> `tours.tour_id`
- `orders.service_id` -> `services.service_id`
- `orders.manager_id` -> `managers.manager_id`

## Почему PostgreSQL подходит
В условии указан MySQL как пример инструмента, но PostgreSQL является полностью подходящим вариантом: требования к проектированию (таблицы, PK/FK, связи, SQL-скрипт) одинаково выполняются в PostgreSQL.
